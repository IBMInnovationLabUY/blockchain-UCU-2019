Programa de fidelización de clientes

Esta aplicacion web es una demo de un programa de fidelización de clientes con blockchain usando Hyperledger Composer API


Un programa de fidelización de clientes permite a las empresas recompensar a los clientes que realizan compras con frecuencia. Los miembros del programa pueden ganar puntos en compras, lo que puede traducirse en algún tipo de recompensa, como descuentos, obsequios o tratamientos especiales para clientes. Los miembros trabajan para obtener una cierta cantidad de puntos para canjear su recompensa. Estos programas pueden tener múltiples compañías como socios en el programa, para atender a una base de clientes. Sin embargo, los sistemas actuales de programas de fidelización son restricciones en las relaciones. entre socios y con visibilidad para los miembros. Estas restricciones se pueden eliminar creando el programa de fidelización del cliente en una red blockchain.

Este modelo de blockchain para un programa de fidelización de clientes mejora el valor de los puntos para los miembros del programa de fidelización y aporta un nuevo valor a los socios mediante la creación de transacciones de confianza. Los participantes en esta red tienen una relación más nivelada entre sí y los puntos están en una posición céntrica para conectar a todos los participantes.

En este patrón de código, crearemos un programa de fidelización de clientes como una aplicación web de blockchain utilizando Hyperledger Composer API y Node.js. La aplicación permitirá a los miembros registrarse en la red donde crearán su cuenta. Se identificarán en la red con su número de cuenta y crearán una clave de acceso que utilizarán para iniciar sesión. Esta clave de acceso se utiliza como la identificación de la tarjeta para que el miembro realice transacciones y consulte registros. El miembro una vez firmado en, puede realizar transacciones para ganar puntos y canjear puntos de los socios en la red. Pueden ver sus transacciones como parte del libro mayor de blockchain. Este patrón de código ilustra el uso de permisos como parte de la red donde un el miembro solo puede ver sus transacciones.

De manera similar para el socio, se registrarán creando una identidad en la red y una clave de acceso que se utilizará para ver sus registros. Los socios pueden ver solo las transacciones de las que formaron parte y, por lo tanto, pueden realizar un seguimiento de todas sus transacciones donde asignaron o canjearon puntos. La aplicación web muestra un panel básico para el socio que muestra los puntos totales que han asignado y canjeado a los miembros. A medida que las transacciones se vuelven complejas, el socio puede realizar análisis en sus transacciones para crear paneles informativos.
